Compare:
- Superpowers if you want the full TDD enforcement and autonomous execution in one tool — the spec is a means to an end (getting clean code), not a standalone artifact.
- Spec Kit if you want the spec itself to be the primary artifact — versioned, reviewed by humans at each phase, portable across tools and agents.
- BMAD if you're running a larger team and want dedicated agent personas (BA, PM, Architect) each owning their domain of the spec before a developer touches anything.
